# 计算机中的数值

在现代计算机中，通常采取32bit or 64bit的存储方式，但在本篇blog中，为了方便学习和演示，尽可能统一采取4bit的表示方式。

### 二进制补码（Two's Complement）：

eg: 

$$
\begin{align*}
1011_2 &= -1\times(1\times2^3) + 0\times2^2 + 1\times2^1 + 1\times2^0 = -5
\end{align*}
$$

*Two's Complement* 与 *Unsigned binary* 的不同就在于最高位的权重正负号。前者可以表示-8\~+7，而后者只能表示0\~16，也就是两者模相同$2^4$。

#### Negate a number

二进制补码有一个有趣且容易推导的性质，即已有数字x_{补码} 想要得到 -x_{补码} , 只需要将各位取反，最后再+1既可。

- 1. *取反* 将原本的数字拆为第一位和其他位的和，a + b, 取反得到 -a + (16 - 1 - b) （这里之所以取反不是  模直接-b， 而多一个-1，因为相当于 111系统取反 而非 1000系统取反）  

- 2. *+1*  +1 得到 M（16）- a - b = -（a + b) (M在该系统中不影响值）


### 原码（Sign-Magnitude)与反码（One's Complement）

原码需要区分码位为符号位和数值位。
- 符号位：0正1负，单独表示正负，不参与数值大小
- 数值为：剩余位置，表示数值大小

原码的问题在于0有两种表示，0000和1000，四位只能表示-7~+7

反码为了解决原码无法加减的问题而存在（本质上是因为符号位不满足加减性质）

反码中的整数表示和原码完全一致，但负数表示则需要数值位按位取反，同样存在0表示重复的问题，四位只能表示-7~+7。

反码的本质是**模运算（Modulo Arithmetic）**，将-x转化为M-x ， 模内的正补数，这样系统中实际就不存在符号位了（所有数实际上都转为了正整数，现在的binary system可以进行加法运算）， $a - x = M + a - x$  这其中的原理是模M系统中，加减M都不影响数值，M≡0。

反码加法中唯一特殊的规则在于 最高位溢出时需要循环进位1，这是由于硬件上按照$2^n$进位（溢出），而实际上我们的反码系统是模$2^n - 1$的。这个问题仅仅在最高位溢出时出现，由于软硬件的模差，而非溢出时，实际上不取错误的模，并不会出现。

硬件上：
$$a + (M - x) - 2^n$$

数学上：
$$a + (M - x) - M$$

而补码很好的解决了软硬件模不一致的问题，不需要进位加一，而由于补码的原理同样是$a - x = M + a - x$，这里$M_{补码}$恰好为 $M_{反码}+1$，因此补码本身的运算就是 反码+1 （仅仅针对负数。
